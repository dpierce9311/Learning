Interactive Training: Click vs Select

Industry Standard: This training aligns with Microsoft, IBM, and Workday style guides. The core principle: use "select" for actions that change a persistent state (user makes a choice), and "click" for immediate actions without persistent state changes.

Use SELECT for:

  • Checkboxes (toggling on/off states)
  • Radio buttons (choosing one option)
  • Dropdown menus (picking from a list)
  • Toggle switches (enabling/disabling)
  • List items (making selections)

Use CLICK for:

  • Submit/Action buttons (Save, Delete, Submit)
  • Navigation links (View Profile, Go to Dashboard)
  • Icons (Download, Print, Close)
  • Tabs (switching views)
  • Modal close buttons
